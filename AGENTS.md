# 宇涧水晶小程序重构规则

## 项目目标

本项目是“宇涧水晶”水晶手串高级定制小程序，包含五行测算、方案推荐、DIY 工作台、灵感社区、购物/下单、订单履约和个人中心。

本轮前端重构的视觉方向为：

Light Studio Lab 明亮高定工作室。

核心特征：

- 暖白空间
- 黑白作为结构主题
- 少量高纯度功能撞色
- 细线条、清晰层级、大留白
- 水晶和天然材质图片保留真实颜色
- 高级、专业、现代，但不能暗黑压抑
- 不做普通商城风、玄学 App 风或大面积渐变风

## 工作方式

- 修改代码前，先识别当前页面的路由、数据来源、样式和依赖组件。
- 每个任务只重构一个指定页面；不得顺手修改其他页面。
- 不得擅自迁移框架或重写项目架构。
- 共享组件或全局样式只有在确有必要时才能修改。
- 修改共享组件后，必须检查已完成页面是否发生视觉或功能回归。
- 不允许删除或弱化原有业务功能。
- 不允许用静态假数据替代真实接口、真实状态或真实跳转。
- 不得修改后端接口、数据字段、登录、支付和订单逻辑，除非任务明确要求。
- 未经确认不得安装新的生产依赖。
- 优先复用现有组件、工具和小程序原生能力。
- 开发新组件、复杂交互或通用能力时，优先调研并复用成熟的 GitHub/npm 开源方案；落地前必须评估许可证、维护状态、包体积、小程序兼容性和现有架构适配性，未经确认不得新增生产依赖。
- 样式必须优先使用统一设计变量，不得在页面中大量散落硬编码颜色和间距。

## 图片素材处理规则

- 后续新增珠子图片时，原图、WPS 抠图和临时修图结果不得直接上传到小程序或 COS，必须先走 `docs/assets/bead-image-pipeline.md` 的统一素材管线。
- DIY 成串圆珠素材必须输出为透明背景、正方形、居中、统一珠体占比的 WebP；前端不得为某一批图片单独写放大/缩小补丁来掩盖素材不统一。
- 珠子 App 素材默认规格为 `512 x 512` WebP，珠体有效直径约占画布 `98% - 99%`，边缘不能有白底、白圈、明显透明留白或偏心。
- 上传前必须生成明暗背景 contact sheet 做人工验收；上传后必须验证 COS 对象、CDN URL、测试环境材料接口和 DIY 工作台成串效果。
- 图片和音频资源默认上传腾讯云对象存储；没有用户明确命令时，只处理和部署测试环境。

## 设计规范

严格遵循：

- `docs/design/design-system.md`
- 当前页面对应的 `docs/design/page-specs/Pxx-xxx.md`
- 当前页面对应的 `docs/design/references/Pxx-xxx.png`

视觉参考图用于还原设计语言，不代表可以牺牲现有功能。

## 验证要求

每次修改完成后必须：

1. 运行项目已有的语法检查、测试或构建命令。
2. 检查页面原有入口、按钮、弹窗、接口和状态是否仍然有效。
3. 检查空状态、加载状态、异常状态和禁用状态。
4. 检查不同手机宽度、状态栏、胶囊按钮和安全区域。
5. 输出修改文件清单、验证结果和仍存在的风险。
6. 不要在未验证的情况下声称任务完成。

## 当前项目命令

- 后端本地启动：`.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000`
- 后端测试：`.\.venv_codex\Scripts\python.exe -m pytest -q`
- Python 语法检查：`.\.venv_codex\Scripts\python.exe -m py_compile <file>`
- 小程序依赖安装：`cd miniprogram && npm install`
- 小程序 JS 语法检查：`node --check <file>`
- 小程序构建/预览：使用微信开发者工具导入 `miniprogram` 目录后编译预览。

## 环境与部署规则

- 没有用户明确命令时，只部署测试环境。
- 上线前再统一打包部署正式环境。
- 本地调试和测试环境切换必须明确说明服务地址与数据库环境。
