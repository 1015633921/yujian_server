# 项目审计提示词

在正式开始 P00 或 P01 前，可先执行一次只读审计。审计任务不得修改文件。

```text
先不要修改任何代码。

请完整审计当前小程序项目，重点回答：

1. 当前使用的技术栈是什么：原生小程序、uni-app、Taro 或其他。
2. 页面路由和底部导航分别在哪里配置。
3. 首页、定制方式页、测算页、推荐页、DIY 工作台、方案确认页、购物车、订单页、灵感页、我的方案页和个人中心分别对应哪些文件。
4. 当前全局样式、主题变量、字体、图标和公共组件分别在哪里。
5. 页面数据来自接口、状态管理还是本地数据。
6. 哪些组件会被多个页面共用。
7. 当前项目的启动、构建、lint、测试命令是什么。
8. 当前页面重构最可能产生功能回归的地方有哪些。

请阅读根目录 AGENTS.md 和 docs/design/design-system.md。

最终只输出：

- 项目结构摘要
- 页面与文件映射表
- 共享组件清单
- 推荐重构顺序
- 风险清单
- 建议补充到 AGENTS.md 的内容

本任务只分析，不修改文件，不安装依赖。
```
