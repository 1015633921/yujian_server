# 页面重构任务模板

后续每个页面任务都按这个模板发起，保持“一页一个任务、一页一次验收”。

## 任务模板

```text
任务编号：
PXX

任务名称：
[页面名称]重构

目标：
本次只重构[页面名称]，使其符合 Light Studio Lab 视觉体系，同时完整保留当前页面的业务功能和真实数据逻辑。

上下文：
- 页面实现：[页面文件路径]
- 页面样式：[样式文件路径]
- 路由配置：miniprogram/app.json
- 参考效果图：docs/design/references/[图片名称]
- 页面规格：docs/design/page-specs/[规格名称]
- 全局规范：docs/design/design-system.md

允许修改：
- 当前页面文件
- 当前页面私有样式
- 当前页面私有组件
- 必要的设计变量或通用组件

禁止修改：
- 其他业务页面
- 后端接口
- 数据字段
- 登录、支付和订单逻辑
- 与本页面无关的路由
- 项目技术栈
- 未经批准的新依赖

必须保留的功能：
[逐条列出所有功能]

视觉要求：
[逐条列出页面结构、颜色、字体、卡片、按钮、图片和状态要求]

交互要求：
[点击、跳转、滚动、弹窗、加载、空状态、禁用状态等]

实施方式：
1. 先检查现有实现。
2. 先给出修改计划和预计涉及文件。
3. 未发现阻塞问题后再实施。
4. 优先复用组件和设计变量。
5. 不要通过隐藏旧功能实现视觉还原。
6. 完成后运行项目已有验证命令。

完成标准：
[可检查、可验证的标准]

最终输出：
- 修改文件清单
- 核心实现说明
- 保留功能清单
- 测试和构建结果
- 未解决风险
- 建议人工检查的位置
```

## 页面验收模板

```text
现在只对 PXX [页面名称] 进行自查和修正，不扩展任务范围。

请依次检查：

1. 原有页面功能是否全部保留。
2. 所有入口是否仍然跳转到原路由。
3. 是否符合 Light Studio Lab 视觉体系。
4. 功能撞色是否只用于状态、编号、图标和重点操作。
5. 是否存在散落的硬编码颜色、圆角和间距。
6. 是否修改了与当前页面无关的页面。
7. 是否影响了底部导航、状态栏、胶囊按钮和安全区域。
8. 加载、空数据、图片失败和异常状态是否合理。
9. 运行已有验证命令。
10. 审查最终 diff，删除无用代码和调试输出。

发现问题请直接修复。

最后输出：
- 最终修改文件
- 功能保留结果
- 验证命令及结果
- 尚需人工确认的项目
```
