# 星灵水晶小程序设计语言

## 1. 品牌气质

关键词：自然疗愈、安静、轻盈、可信赖、可触摸的水晶质感。

视觉避免：

- 大面积高饱和色和纯黑按钮。
- 小圆角、粗边框、密集阴影。
- 同一页面混用多套标题、按钮和卡片风格。
- 用装饰抢占内容层级。

## 2. 色彩

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| 页面底色 | `#F8F6F1` | 页面背景 |
| 高亮底色 | `#FBFAF7` | 导航、浅色区域 |
| 卡片 | `rgba(255,255,255,.72)` | 内容卡片、表单 |
| 主文字 | `#303230` | 标题、正文 |
| 次文字 | `#858782` | 描述、辅助信息 |
| 主品牌色 | `#929E87` | 主按钮、选中态 |
| 深品牌色 | `#626C5D` | 图标、强调文字 |
| 浅品牌色 | `#EEF0E9` | 标签、选中背景 |
| 砂金点缀 | `#A99577` | 少量价格、星光和品质感 |
| 分割线 | `rgba(140,145,133,.13)` | 卡片和列表分隔 |
| 错误色 | `#A85B4A` | 删除、错误提示 |

主色使用规则：单屏主品牌色面积不超过约 20%，主要用于按钮、选中态和关键状态。

## 3. 字体与层级

- 品牌标题、页面大标题、章节标题：楷体气质字体，优先 `STKaiti / KaiTi`。
- 正文、按钮、数据：系统无衬线字体。
- 页面大标题：`40–46rpx`。
- 区块标题：`30–34rpx`。
- 卡片标题：`26–30rpx`。
- 正文：`23–26rpx`，行高 `1.5–1.65`。
- 辅助文字：`19–22rpx`。
- 避免使用 `900` 的过重字重；常规标题使用 `500–700`。

## 4. 间距

基础单位为 `8rpx`：

- 页面左右边距：`28–36rpx`。
- 区块间距：`28–36rpx`。
- 卡片内边距：`22–30rpx`。
- 元素小间距：`8–14rpx`。
- 元素常规间距：`16–22rpx`。

## 5. 圆角与阴影

- 大卡片：`24–30rpx`。
- 普通卡片和输入框：`16–24rpx`。
- 按钮、标签：`999rpx` 胶囊形。
- 底部抽屉：顶部 `28–34rpx`。
- 阴影统一轻柔：`0 12rpx 32rpx rgba(76,73,65,.055)`。

## 6. 组件规范

### 主按钮

- 高度 `82–88rpx`。
- 鼠尾草绿渐变。
- 白字，字重 `600`。
- 页面内最多一个主按钮占据最高视觉优先级。

### 次按钮

- 透明暖白背景。
- 鼠尾草绿细边框。
- 与主按钮同为胶囊形。

### 卡片

- 暖白半透明背景。
- 细绿色中性边框。
- 不使用厚重描边。
- 信息卡优先靠留白分组。

### 输入框

- 高度 `78–86rpx`。
- `16–20rpx` 圆角。
- 背景使用 `#F4F4EF` 或半透明白。
- 聚焦和选中使用鼠尾草绿，不使用棕色。

### 标签

- 仅用于分类和轻状态，不替代按钮。
- 浅鼠尾草底、深鼠尾草文字。

### 底部操作栏

- 暖白半透明背景，可加轻微模糊。
- 主操作靠右，占更大宽度。
- 必须预留 `safe-area-inset-bottom`。

### 弹窗

- 优先使用底部抽屉。
- 顶部圆角 `28–34rpx`。
- 标题、说明、表单、操作按钮分区清晰。

## 7. 页面模板

### 浏览列表页

页面标题 → 简介/筛选 → 卡片列表 → 空状态。

### 内容详情页

视觉主图 → 标题和标签 → 分章节内容 → 固定底部操作。

### 表单流程页

任务说明 → 分组表单卡片 → 可选项 → 单一主操作。

### 工具工作台

核心操作画布优先 → 辅助工具弱化 → 材料列表 → 固定结果操作。

### 订单与状态页

状态最先展示 → 地址/材料/物流分卡片 → 与当前状态相关的操作。

## 8. 复用方式

通用类已放入 `miniprogram/app.wxss`：

- `.page`
- `.section`
- `.section-head`
- `.section-title`
- `.section-link`
- `.card`
- `.primary-button`
- `.secondary-button`
- `.tag`
- `.nav-row`
- `.nav-title`
- `.icon-btn`
- `.design-page-heading`
- `.design-eyebrow`
- `.design-title`
- `.design-desc`

新页面应先使用这些基础类，仅在页面 WXSS 中补充业务布局，避免重新定义颜色、按钮和卡片体系。
